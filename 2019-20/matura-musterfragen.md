# Themenbereiche NVS MC

## Hinweise und Tipps

* Die unten angeführten Fragen sind *Musterfragen*. Bei der Matura können die hier gestellten Fragen in abgewandelter Form (z.B. anders kombiniert, mit oder ohne Beispiel etc.) gestellt werden. Inhaltlich werden die Maturafragen aber die Themenbereiche der Musterfragen abdecken.
* Beantworten Sie die gestellten *Fragen*. Vermeiden Sie es, bei der Matura von sich aus ungefragt auf andere Themenbereiche einzugehen.
* Erklären Sie abgefragte Konzepte anhand von Beispielen oder Diagrammen (z.B. Architekturdiagramm, Ablaufdiagramm, Sequenzdiagramm) wenn das inhaltlich Sinn macht. Codebeispiele müssen nicht syntaktisch vollständig korrekt sein, müssen die Grundkonzepte aber nachvollziehbar widergeben und dürfen keine schweren Fehler enthalten.

## Ausarbeitung

* Jede Klasse muss Antworten auf die unten angeführten Musterfragen ausarbeiten. Die Dokumentation der Ausarbeitungen erfolgt auf **GitHub oder GitLabs** (kann je Klasse gewählt werden).
* Jede Klasse bestimmt eine Administratorin oder einen Administrator, der das entsprechende Repository in GitHub oder GitLabs anlegt und eventuell notwendige Berechtigungen erteilt. **Hr. Stropek muss Leserechte auf das Repository erhalten.**
* Jede Klasse verteilt die Themengebiete (d.h. Musterfragen) auf alle Schülerinnen und Schüler.
  * Je Themengebiet gibt es eine Person, die die initiale Ausarbeitung übernimmt und sie eincheckt.
  * Je Themengebiet muss es zwei Personen geben, die die Qualitätssicherung übernehmen. Sie geben der ausarbeitenden Person Feedback über Verständlichkeit, Vollständigkeit und Richtigkeit der Ausarbeitung.
  * Der Administrator ist dafür verantwortlich, im Repository eine Übersicht über alle Themengebiete (d.h. Musterfragen) zu erstellen und dort die ausarbeitende Person und die Qualitätssicherer zu nennen. **Tipp**: Ausarbeitungen in der Übersicht verlinken.
* Je Themengebiet (d.h. Musterfrage) muss es im Repository eine **Markdown**-Datei mit der Ausarbeitung geben. Kriterien:
  * Stichwortartige Beantwortung ist ausreichend, ausformulierter Text ist nicht notwendig.
  * Wo gefordert bzw. sinnvoll muss die Ausarbeitung Codebeispiele enthalten.
  * Wo gefordert bzw. sinnvoll muss die Ausarbeitung Diagramme (z.B. Architekturdiagramm, Sequenzdiagramm) enthalten.
  * Anhand der Git-Historie muss ersichtlich sein wer wann zu welcher Ausarbeitung beigetragen hat.
